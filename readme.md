**Web Server的搭建（附测试客户端）**

*By 刘一凡*
- 运行步骤
    - 运行main.py，配置一个有效的端口（为了方便，建议您以80作为配置）
    - 看到'Server Thread ready to activate, waiting for connections.'提示后，
    您可以：
        - 运行Client.py,输入您刚才的端口配置。（注意，输错端口不能连接）
        - 打开您的浏览器，复制main.py给出的网址
          (在go ahead and visit后面)并粘贴到您的浏览器地址栏中进行浏览
    - 在您的浏览器/Client.py中可以看到对应信息，这也可以从main.py的输出
    同步看到。
- 完成要求：
1. 网络服务器能够正确地对客户端（浏览器和Client.py)的请求进行回应，
   且能进行一些异常处理：
   - 遇到文件请求失败时（如在浏览器中请求不存在的文件）会返回404 Not Found报文
   - 在客户端强行断开连接时也能在控制台给出提示，放弃该连接并恢复工作
   - 客户请求报文不合规范时返回400 Bad Request报文
2. 网络服务器使用多线程操作，可以为两个HTTP请求同时服务
3. 用户可以在网络服务器初始化时手动指定端口号，若跳过该步骤则默认为80
